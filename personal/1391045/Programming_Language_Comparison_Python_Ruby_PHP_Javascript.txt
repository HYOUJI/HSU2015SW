1. Python
- 변수를 선언할 때 변수타입을 따로 선언할 필요가 없어 변수명만 적어주면 되기 때문에 편리하다. 그렇기 때문에 프로그래밍 시간이 절약된다.
- dictionary 라는 자료구조를 제공하는데, 처음 선언할 때 순서를 아무렇게나 해도 keys값의 알파벳 순으로 정렬이 된다. del 을 이용하여 삭제할 수 도 있어서 편리했다.
- ;(세미콜론)과 {, }(중괄호)가 필요 없이 오로지 들여쓰기로 구분한다. 들여쓰기를 잘못하면 실행이 되지않는다.
- 문자열과 리스트를 위한 다양한 method가 있다. 
- 배우기가 쉬운 언어이다. 다른 언어들을 하고 마지막으로 Python을 하였는데 Python부
터 했으면 더 좋았을 것 같다. Python을 마지막으로 해서 그런지 Ruby와 비슷한 느낌이 들었다. 


2. Ruby
- ;(세미콜론)을 안적어도 된다. 그러나 end로 함수나 조건문 등의 끝을 나타낸다. 그렇기 때문에 끝을 알 수 있어서 좋다.
- 변수를 선언할 때 변수타입을 따로 선언할 필요가 없어 변수명만 적어주면 되기 때문에 편리하다.
- 다양한 반복문이 있어서 상황에 맞게 간결하게 쓸수 있어서 좋았다.
- 같은 스크립트 언어라서 그런지 Python과 비슷한 느낌이었다. Python과 마찬가지로 Ruby또한 배우기 쉬웠던거같다.
- 배열을 선언할 때 배열에 다양한 타입의 객체를 넣을 수 있고, 배열에 원소를 추가하거나 삭제 할 수 있다.

3. PHP
- 웹을 개발할 때 쓰이는 언어이기 때문에 HTML에 쓰일때 <?php (코드)  ?> 태그를 통해 PHP를 한눈에 구분 할 수있다.
- 변수 앞에 $를 붙여서 선언하는데, 변수타입이 따로 필요하지않다.
- 함수를 만들때 function 이라고 써준뒤 함수명을 써서 함수인 것을 한눈에 알아보기 쉽다. 
- PHP의 문법은 C, Java, Perl 언어를 많이 참조하였고, C언어와 유사하여 배우기 쉽다.
- 다양한 데이터베이스를 지원하고 자체 라이브러리를 지원하여 연결이 쉽다.

4. Javascript
- 모든 변수타입 및 함수의 선언은 var로 되어있어 편리하다. 
- C, Java와 문법이 비슷하여 배우는데 크게 어렵지 않았다.
- console.log(), alert(), prompt()등을 이용하여 입출력을 다양한 방법으로 할 수 있다. (HTML로만 할수 없는 것들을 할 수 있게 해준다.)
- 함수를 만들때 function이라고 써준뒤 함수명을 쓰기 때문에 함수인 것을 한눈에 알아보기 쉽다.
- 컴파일 과정이 필요없고 웹 브라우저에서 바로 실행을 할 수 있다.
